Stage 2 ¬∑ Set Up Dataverse Tables üóÑÔ∏è

Configure the Dataverse tables GroundCrew.AI will use to store onboarding data and track Alex's journey.

Mission Checklist

Create the UserVerification table in Power Apps with the columns needed to track email verification, headshot status, and OTP codes for Alex's pre-boarding journey.

Access Power Apps
1

Go to https://make.powerapps.com.

Sign in with your demo credentials. You should be brought to the page shown below.

Verify Environment
2
‚ö†Ô∏è Important: Ensure you are in the right environment. Check the top right ‚Äî you should be in Contoso (Default).

If you're not in the correct environment, select it from the environment dropdown in the top right corner. You might be prompted to create a new developer environment ‚Äî if so, click OK and take note of the environment name on the top right after your page refreshes.

Create New Table
3

On the left-hand side panel, select 'Tables', then click 'New Table'.

üö® IMPORTANT: When you click "New Table", you might be prompted to create a new developer environment. Click OK to proceed. After the page refreshes, take note of the environment name displayed in the top right corner ‚Äî this is the environment you'll be using in Stage 3 when building the agent in Copilot Studio. Make sure you're working in the same environment for both stages!

On the 'choose an option to create tables' page, choose to 'Start from blank'.

Configure Table Properties
4

Click the tab on the top right of your 'Table 1' block and click 'View properties'.

Edit the Display name to "UserVerification". The plural name should automatically be "UserVerifications". Then click 'Save'.

Edit the First Column
5

On the table designer, click on 'New column' and choose to edit column. We will edit the first column that is auto-created for us, then create the remaining columns.

Edit the first column with these details and click 'Update':

Column Name Data Type Required
Name Single line of text Yes
Add Remaining Columns
6

Click '+ New column' and add 5 new columns with the following details:

Column Name Data Type Required
Email Single line of text Yes
Verification Status Single line of text Off
Headshot Status Single line of text Off
OTP code Whole Number Off
Headshot File > Image Off

Once you have all 6 columns, you may "Save and exit".

Add Test Data
7

Add your name and an email address that you can access in the Name and Email columns.

This test data will be used when testing the agent flows. Make sure you use an email address you can access, as it will be used for verification in later stages.
Mission Complete
8

Congratulations! You have created the table that is going to be used in building our agent and its flows!

This UserVerification table will store Alex's onboarding data, track verification status, headshot uploads, and OTP codes throughout the pre-boarding journey.

Once the table is saved, you're ready to move to Stage 3 to build the agent that will use this table.
