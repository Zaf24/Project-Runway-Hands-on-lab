Stage 3 ¬∑ Build the Agent üõ†Ô∏è

Spin up GroundCrew.AI inside Copilot Studio, seed it with knowledge, and prep the conversational brain that will greet Alex before Day 1.

Mission Checklist

Create the agent, describe its mission, connect SharePoint knowledge, and test that it can answer onboarding questions with confidence.

Change Your Environment
1
üö® Highly important step: In the Microsoft Copilot Studio page, ensure you change your environment to the same environment as the one you created your Power Apps table in.

This ensures that GroundCrew.AI can access the Dataverse tables you set up in Stage 2. Verify your environment selection matches before proceeding.

Launch a New Agent
2

In the Microsoft Copilot Studio page, click on the "Create" button at the taskbar on the left side, then click "New agent". This opens the agent creation flow.

Describe GroundCrew.AI
3

On the "Describe" tab, enter this prompt so Copilot seeds the right structure:

"I want to create a pre-onboarding agent that would assist a new hire with activities such as email verification and headshot uploading. The agent should also be able to answer any HR or pre-onboarding questions from the knowledge base"

Copilot should build the basics for the agent for you as shown in the image. Give it a name of your preference or just say "ok" to the suggested name.

After you are done, click the 'Create' button on the top right.

Add SharePoint Knowledge
4

Click "Knowledge" on the top taskbar.

Click "Add Knowledge".

Choose to add from SharePoint. On the pop-up window, choose to 'browse items'.

You will be brought to this page. Click on 'The Landing' and select JTC AI Demo folder. Select all 3 files in the folder and click "Confirm selection" and "Add to agent".

You should see that all your files are added to the knowledge and ready to be queried!

Test the Agent
5

Test your agent on the test pane on the right side of your window. Use "What are the onboarding processes I can expect" or any other prompt of your choice!

Try a mix of questions about schedules, dress code, and pre-boarding paperwork to ensure the agent uses the uploaded files correctly.

Well done! You have now created an agent and added knowledge base to it, enabling it to answer any user query in a conversation!

Once the responses look accurate and helpful, you're clear to move to Stage 4 to configure the conversation start topic and enable general enquiries using conversational boosting.
